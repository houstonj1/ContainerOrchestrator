Using Container Orchestrator:

Pulling Images:

Container Orchestratator is able to pull any image hosted on Docker's public image registry. Once an image is pulled onto the local system, it can be used to create containers. Go to the Images page to pull an image. For more information on pulling Docker images, visit Docker's tutorial on exploring and pulling images. To explore all available images, go to Docker's public image registry, Docker Hub.

Creating Containers:

After pulling an image, you can start creating containers! Visit the Containers page to start creating. Container Orchestrator supports configuring the container command, as well as network settings. You can view container information by clicking on one of the listed containers. For more information on containers and container configuration, visit Docker's documentation.

Docker Errors:

When working on Container Orchestrator, it may be useful to understand some Docker errors in order to understand alerts that may be experienced while using this application. Overall, there are four important errors that the user may experience:

  • Docker APIError

    • The Docker APIError occurs when there is an error thrown by the Docker Engine API. This acts as a RESTful API, returning 404, 409 errors when invalid information is passed, or if the request cannot be completed

  • Docker ImageNotFound

    • The Docker ImageNotFound error is returned only during operations with or on images. For example, if a user tries to create a container from an image not on the system, it will thrown this error. Similarly, this error will be thrown if the user tries to pull an image that does not exist on Docker Hub

  • Requests ConnectionError
  • Requests ConnectTimeout

    • Both of these requests errors are related to the Docker Engine API, which as mentioned above is a RESTful API. These errors will be thrown if the Docker Engine is unable to make requests to the API or if the requests time out.

More About the Docker Engine:

Docker Engine API:

The Docker Engine is the software that allows for the creation and usage of containers. Container Orchestrator uses the Docker Engine API to control and orchestrate containers on this system. For more information on the Docker Engine and the Docker Engine API, visit Docker's documentation.
